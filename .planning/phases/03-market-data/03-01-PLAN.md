<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Express API | User-supplied symbol/coinId parameters in URL path and query string |
| Express -> Yahoo Finance | Server fetches from unofficial Yahoo API with user-supplied symbol |
| Express -> CoinGecko | Server fetches from CoinGecko API with user-supplied coinId |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | /market/stocks/:symbol | mitigate | Validate symbol with regex `/^[A-Za-z0-9.]{1,10}$/` before passing to yahoo-finance2. Reject with 400 on invalid input. |
| T-03-02 | Tampering | /market/crypto/:id/chart | mitigate | Validate coinId with regex `/^[a-z0-9-]{1,50}$/`. Reject with 400 on invalid input. |
| T-03-03 | Tampering | Upstream API responses | mitigate | Normalize and pick only expected fields in service layer before caching. Never pass raw upstream JSON to client. |
| T-03-04 | Denial of Service | All /market/* routes | mitigate | Existing express-rate-limit (100 req/15 min) on /api/* prefix. Server-side cache prevents upstream API abuse regardless of client request volume. |
| T-03-05 | Information Disclosure | Error responses | mitigate | Return generic error messages to client. Do not leak upstream API error details or stack traces in production (existing error handler checks NODE_ENV). |
| T-03-06 | Spoofing | CoinGecko image URLs | accept | Coin images (c.image) are URLs from CoinGecko CDN. CSP imgSrc should add `*.coingecko.com` in Plan 02 when rendering images. Low risk: images are display-only. |
</threat_model>
