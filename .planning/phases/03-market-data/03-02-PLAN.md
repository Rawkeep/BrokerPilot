<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> StockSearch | User types arbitrary text as stock symbol |
| API response -> Chart rendering | Server data rendered into canvas via lightweight-charts |
| CoinGecko image URLs -> img tags | External image URLs rendered in CryptoTable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | StockSearch input | mitigate | Validate symbol client-side with `/^[A-Za-z0-9.]{1,10}$/` before sending to API. Server also validates (Plan 01 T-03-01). |
| T-03-08 | Spoofing | CoinGecko coin images in CryptoTable | mitigate | CSP imgSrc updated to allow `https://*.coingecko.com`. Images rendered via `<img>` tag with alt text. No script execution risk from img tags. |
| T-03-09 | Denial of Service | useMarketData polling | accept | 60-second poll interval is reasonable. No polling for chart data (on-demand only). Server-side cache and rate limiter prevent upstream abuse. |
| T-03-10 | Information Disclosure | Error messages in UI | mitigate | Display generic German error text from i18n, not raw API error details. Error details logged to console only in development. |
</threat_model>
