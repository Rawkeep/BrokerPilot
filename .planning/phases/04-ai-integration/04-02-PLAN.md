       - partialize: only persist selectedProvider and selectedModel (not transient UI state)
       - Include a DEFAULT_MODELS map: { anthropic: 'claude-sonnet-4-20250514', openai: 'gpt-4o-mini', google: 'gemini-2.0-flash', mistral: 'mistral-small-latest', groq: 'llama-3.1-8b-instant', openrouter: 'meta-llama/llama-3.1-8b-instruct:free' }
       - When setSelectedProvider is called, also auto-set selectedModel to DEFAULT_MODELS[provider]

    2. Create client/src/services/aiApi.js:
       - Follow the same pattern as marketApi.js
       - Export async function sendAIRequest(provider, model, messages, apiKey = null):
         a. Build body: { provider, model, messages }. If apiKey is truthy, add apiKey to body.
         b. POST to /api/ai/chat with Content-Type: application/json
         c. If response not ok, parse error body and throw Error with the error message
         d. Return parsed JSON response: { content, model, provider, usage }
       - CRITICAL: apiKey is passed in the request body (HTTPS encrypted in transit), never in URL or query params

    3. Create client/src/hooks/useAIRequest.js:
       - Import useAIStore, useKeyStore
       - Export function useAIRequest():
         a. Read selectedProvider, selectedModel from aiStore
         b. Read decryptedKeys, sessionUnlocked from keyStore
         c. Read isLoading, error, lastResponse from aiStore
         d. Provide sendMessage(userMessage: string) async function:
            - Get apiKey = decryptedKeys[selectedProvider] || null (if session not unlocked or no key for provider, apiKey is null — freemium mode)
            - Set isLoading true, error null
            - Build messages array: [{ role: 'user', content: userMessage }]
            - Call sendAIRequest(selectedProvider, selectedModel, messages, apiKey)
            - On success: setLastResponse(response), setIsLoading(false)
            - On error: setError(err.message), setIsLoading(false)
         e. Return { sendMessage, isLoading, error, lastResponse, selectedProvider, selectedModel, hasApiKey: !!decryptedKeys[selectedProvider] }

    4. Write test for useAIRequest hook (tests/hooks/useAIRequest.test.js):
       - Mock aiApi.js sendAIRequest
       - Test: sendMessage calls sendAIRequest with correct provider, model, messages, and apiKey from keyStore
       - Test: sendMessage without API key sends null apiKey (freemium mode)
       - Test: loading state transitions: false -> true -> false
       - Test: error state set on failure
       - Test: lastResponse set on success

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> useAIRequest hook | User's message content is untrusted |
| keyStore decryptedKeys -> API request body | Decrypted API key flows from memory to fetch body |
| /api/ai/chat response -> AIResponsePanel | AI-generated content rendered in UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Information Disclosure | apiKey in fetch body | mitigate | Key sent only in POST body over HTTPS (encrypted in transit). Never logged client-side. Never stored outside keyStore.decryptedKeys (memory-only, cleared on tab close) |
| T-04-10 | Tampering | AI response content rendered as HTML | mitigate | AIResponsePanel renders AI content as text (not dangerouslySetInnerHTML). Use CSS white-space: pre-wrap for formatting. No raw HTML rendering |
| T-04-11 | Information Disclosure | selectedProvider/model in localStorage | accept | Provider and model preferences are not sensitive. Persisted for UX convenience. API keys are NOT in this store |
| T-04-12 | Denial of Service | rapid message sending | mitigate | Send button disabled during isLoading. Backend rate limiter (100 req/15min) and freemium gate provide server-side protection |
</threat_model>
