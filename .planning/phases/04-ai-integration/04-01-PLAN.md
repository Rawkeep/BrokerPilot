<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Express /api/ai/chat | Untrusted input: provider, model, messages, apiKey all from client |
| Express -> AI Provider APIs | Server relays request with user's API key to external services |
| AI Provider -> Express | Untrusted AI output: content could contain injection attempts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | freemiumGate IP tracking | mitigate | Use req.ip (respects X-Forwarded-For on Railway). Accept that shared IPs may unfairly limit users — acceptable for freemium |
| T-04-02 | Tampering | req.body.apiKey | mitigate | Validate apiKey is a string, pass through to provider only. Never log, persist, or echo back. Zod validates request shape |
| T-04-03 | Repudiation | AI requests | accept | No user auth in v1 — anonymous usage. Logging request metadata (provider, model, IP) without apiKey is acceptable |
| T-04-04 | Information Disclosure | apiKey in error logs/responses | mitigate | Never include apiKey in error responses or console.log. costGuard and circuit breaker errors omit key. Server error handler already sanitizes in production |
| T-04-05 | Denial of Service | token budget / runaway requests | mitigate | costGuard enforces per-request token budget (AI_TOKEN_BUDGET=8000). Circuit breaker stops hammering failing providers. Express global rate limiter (100 req/15min) already active |
| T-04-06 | Elevation of Privilege | prompt injection in messages | accept | AI proxy is a relay — no server-side agent logic in Phase 4. Prompt injection affects the user's own AI session only. Phase 5 agents will add Zod output validation for structured data |
| T-04-07 | Information Disclosure | AI response content (XSS) | mitigate | Zod validates response structure. Frontend (Phase 4 Plan 02) must sanitize AI content before rendering as HTML. Server returns JSON only |
| T-04-08 | Denial of Service | freemium gate memory growth | mitigate | IP counter Map entries expire at midnight UTC. Stale entries cleaned on access. Bounded by unique IPs per day — acceptable for v1 scale |
</threat_model>
