<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/agents/run | Untrusted user input (agentType, payload, apiKey) crosses to server |
| server -> AI provider | User's BYOK key + constructed prompt sent to external API |
| AI provider -> server | AI response (untrusted content) parsed as JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | agentRouter | mitigate | Validate request body with AgentRunRequestSchema (Zod) at route entry — reject unknown fields |
| T-05-02 | Tampering | payload data | mitigate | Sanitize lead/deal data in buildPrompt — do not pass raw user HTML/script into prompts |
| T-05-03 | Information Disclosure | SSE stream | mitigate | Never include apiKey in SSE events — only agent results and status |
| T-05-04 | Denial of Service | agent execution | mitigate | Reuse existing freemiumGate (5/day) and circuit breaker per provider; AI_REQUEST_TIMEOUT (30s) prevents hanging |
| T-05-05 | Elevation of Privilege | prompt injection | accept | User controls their own data and BYOK key — prompt injection affects only their own AI responses, not server state |
| T-05-06 | Tampering | AI response parsing | mitigate | Zod schema validation on all AI output — reject responses that don't match expected structure |
</threat_model>
