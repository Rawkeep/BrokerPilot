    - Helper `formatDate(isoStr)`: DD.MM.YYYY format

Map: { 'Name': 'name', 'E-Mail': 'email', 'Telefon': 'phone', 'Unternehmen': 'company', 'Deal-Wert': 'dealValue', 'Phase': 'stage', 'Prioritaet': 'priority', 'Tags': 'tags', 'Erstellt': 'createdAt' }. Also map broker-specific field labels to keys using BROKER_TYPES.
    - `parseGermanNumber(str)`: Remove dots, replace comma with dot, parseFloat
    - `mapCSVToLeads(rows, columnMapping, brokerType)`:
      1. For each row: create base lead via createLead(brokerType)
      2. Map columns using columnMapping
      3. Parse German numbers for currency/number fields
      4. Split tags by comma
      5. Validate with validateLead()
      6. Return `{ valid: Lead[], invalid: { row, errors }[] }`

| T-07-02 | Injection | CSV import | mitigate | PapaParse handles quoting/escaping; never eval() CSV content; field values stored as plain strings |
| T-07-03 | Information Disclosure | CSV export | accept | CSV contains user's own lead data exported from their browser — no cross-user data exposure possible |
| T-07-04 | Injection | PDF generation | mitigate | jsPDF text() method treats content as plain text, not HTML — no XSS vector in PDF; do not use html() method for AI content |
