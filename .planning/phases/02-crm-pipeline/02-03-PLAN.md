|----------|-------------|
| URL params -> lead lookup | leadId from URL used to find lead in store |
| Note input -> activity array | User text added to lead activities |
| Tag input -> lead tags | User-created tags stored on leads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | URL leadId parameter | mitigate | Validate leadId exists in store before rendering. Show "nicht gefunden" for invalid IDs. |
| T-02-08 | Denial of Service | Tag/note input flooding | mitigate | Tags capped at 10 per lead. Activities capped at 100 (from Plan 02-01 addActivity). Notes max 5000 chars. |
| T-02-09 | Tampering | Search input | accept | Search is client-side filter on user's own data. No injection risk. React escapes rendered values. |
