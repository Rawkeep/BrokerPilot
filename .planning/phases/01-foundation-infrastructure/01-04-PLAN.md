<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User PIN input -> CryptoService | PIN entered in GlassInput flows to crypto operations |
| React Router -> page components | Route parameters could be manipulated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | KeyManager PIN input | mitigate | PIN input type="password" prevents shoulder surfing. No PIN stored anywhere |
| T-04-02 | Tampering | Route manipulation | accept | All routes lead to read-only placeholder pages. No data mutation via routing |
| T-04-03 | Information Disclosure | API key in form input | mitigate | API key input type="password". Key encrypted immediately on save, form cleared |
</threat_model>
