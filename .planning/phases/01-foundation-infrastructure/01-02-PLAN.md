<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User PIN -> CryptoService | PIN is the only secret protecting API keys |
| Memory -> IndexedDB | Encrypted ciphertext crosses to persistent storage |
| IndexedDB -> Memory | Decrypted keys exist only in JS memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | IndexedDB stored keys | mitigate | AES-256-GCM encryption with PBKDF2 600K iterations. Only ciphertext+salt+IV persisted, never plaintext (D-14) |
| T-02-02 | Elevation of Privilege | PIN brute force | accept | PBKDF2 600K iterations makes each attempt ~200ms. No lockout in v1 -- add in v2 if needed. Low risk: attacker needs physical device access |
| T-02-03 | Information Disclosure | Decrypted keys in memory | mitigate | Keys cleared on tab close via beforeunload. partialize excludes decryptedKeys from Zustand persist |
| T-02-04 | Tampering | Encrypted key modification | mitigate | AES-GCM provides authenticated encryption -- modified ciphertext fails decryption |
</threat_model>
