  <action>
    Create `client/src/stores/pipelineRunStore.js` — Zustand store (with persist to localStorage key 'bp-pipeline-runs'):
    - State: `{ runs: [], activePipelineId: null }`
    - Run shape: `{ pipelineId, leadId, leadName, brokerType, status: 'running'|'done'|'partial'|'error', steps: { qualifier: { status, result, error }, analyst: { status, result, error }, swot: { status, result, error } }, createdAt, duration }`
    - Actions: `startPipeline(pipelineId, leadId, leadName, brokerType)`, `updateStep(pipelineId, step, status, data)`, `completePipeline(pipelineId, isPartial, duration)`, `getPipelinesByLeadId(leadId)`
    - Only persist `runs` array, not `activePipelineId`

    Create `client/src/hooks/usePipelineStream.js`:
    - Follow the same fetch + ReadableStream SSE pattern from `useAgentStream.js`
    - `usePipelineStream()` returns `{ triggerPipeline(leadData, brokerType, leadId), isRunning, currentStep, stepStatuses, error }`
    - `triggerPipeline()`:
      1. Read provider/model/apiKey from aiStore and keyStore (same as useAgentStream)
      2. POST to `/api/agents/pipeline` with `{ leadData, brokerType, provider, model, apiKey }`
      3. Parse SSE events from ReadableStream
      4. On `pipeline:start`: init step statuses (all 'pending')
      5. On `pipeline:step:start`: update that step to 'running', set currentStep
      6. On `pipeline:step:done`: update step to 'done' with result, store in pipelineRunStore
      7. On `pipeline:step:error`: update step to 'error' with message
      8. On `pipeline:done`: mark pipeline complete, set isRunning=false
    - Include AbortController cleanup for unmount
    - Prevent double-trigger while isRunning
  </action>
