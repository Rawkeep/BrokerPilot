    - Export a `createInitialState(leadData, brokerType, aiConfig)` factory that returns the pipeline state object: `{ leadData, brokerType, aiConfig, qualifierResult: null, analystResult: null, swotResult: null, completedSteps: [], failedSteps: [], isPartial: false }`

    - Import existing agent modules (leadQualifier, marketAnalyst, swotStrategist) and `relayAIRequest`

    - Create `analystNode(state, emit)`: same pattern with marketAnalyst (includes enrichData call). For the analyst, derive symbol/assetType from leadData.customFields or use a sensible default.

      3. Execute qualifierNode -> analystNode -> swotNode sequentially
      4. Emit `pipeline:done` with final results
      5. Return `{ qualifierResult, analystResult, swotResult, completedSteps, failedSteps, isPartial }`

    Note: While the research mentions LangGraph StateGraph, the actual execution is simpler with a sequential runner that wraps each step in try/catch. Use LangGraph's `StateGraph` + `Annotation` if the import works cleanly with the project's ESM setup. If LangGraph.js has ESM compatibility issues (common pitfall), fall back to a manual sequential executor that provides the same interface. The tests should validate behavior, not implementation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Pipeline endpoint | Untrusted input: leadData, brokerType, apiKey |
| Pipeline -> AI providers | BYOK apiKey forwarded per-request, never stored |
| SSE events -> Client | Must not leak apiKey or server internals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | Pipeline endpoint | mitigate | Validate request body with Zod PipelineRunRequestSchema at route entry |
| T-06-02 | Information Disclosure | SSE events | mitigate | Never include apiKey in SSE event data — emit callback strips sensitive fields |
| T-06-03 | Denial of Service | Pipeline execution | mitigate | Freemium gate counts pipeline as 3 requests; circuit breaker halts on provider failure; 30s timeout per AI call from aiProxy |
| T-06-04 | Elevation of Privilege | Pipeline payload | mitigate | leadData is passed to agent prompts as JSON.stringify — existing sanitization in each agent's buildPrompt applies |
| T-06-05 | Tampering | Pipeline state | accept | Pipeline state is server-side only, not exposed to client. Client cannot modify mid-pipeline state |
</threat_model>
